Documentation

Complete reference for the sRNA-TAPS pipeline — installation, configuration, pipeline steps, benchmarking hooks, and output formats.

Installation and setup

Start with conda, pip, or source installation, then move directly into project configuration and quick-start execution.

Pipeline internals

Each stage is documented in order, from genome setup and alignment through calling, background correction, and reporting.

Outputs and validation

Expected files, delta-score logic, SNP filtering, and test-dataset checkpoints stay close for practical troubleshooting.

Installation

Conda (recommended)

Installs sRNA-TAPS and all bioinformatics dependencies automatically.

conda install -c bioconda -c conda-forge srna-taps

pip

pip install sRNA-TAPS

From source

git clone https://github.com/HenzelerB/sRNA-TAPS
cd sRNA-TAPS
pip install -e .

Full environment

conda env create -f environment.yaml
conda activate RNA_taps

R packages

Required for report figure generation. Figures are produced automatically by the pipeline — install R packages once before the first run.

Rscript install_R_packages.R

Quick Start

sRNA-TAPS runs as a Snakemake 9 profile-based workflow. All SLURM resource allocation is handled by the profile — no cluster submission strings or job scripts to manage.

# 1. Create a project directory and add raw FASTQs
mkdir ~/my_project && cd ~/my_project
mkdir rawfiles
# copy *.fq.gz into rawfiles/

# 2. Create config.yaml and samples.tsv (see Configuration below)

# 3. Dry-run to verify the DAG
snakemake \
    --snakefile /path/to/sRNA-TAPS/workflow/Snakefile \
    --configfile config.yaml \
    --profile    /path/to/sRNA-TAPS/profile \
    --dry-run

# 4. Run the full pipeline
bash /path/to/sRNA-TAPS/sRNA_taps.bash
#    — or directly in the background —
snakemake \
    --snakefile /path/to/sRNA-TAPS/workflow/Snakefile \
    --configfile config.yaml \
    --profile    /path/to/sRNA-TAPS/profile &

# 5. Run with benchmarking (optional)
snakemake \
    --snakefile /path/to/sRNA-TAPS/workflow/Snakefile \
    --configfile config.yaml \
    --profile    /path/to/sRNA-TAPS/profile \
    all_benchmark
Genome auto-download: If the genome FASTA or GTF are absent from the paths in config.yaml, the pipeline downloads GRCh38 from Ensembl and builds the Bowtie1 index automatically. No pre-built index is needed for new users.

samples.tsv

Tab-separated, one row per sample. Four columns are required:

ColumnDescription

conditionExperimental condition — any name; must include a treat, pb_Ctrl, and no-treat equivalent
cell_lineCell line or biological source — any name; used to group SNP blacklists and colour palettes
fastqFull path to the input .fq.gz file (single-end)

Pipeline Overview

rawfiles/         Raw FASTQs (SE, TruSeq small RNA)
    ↓
    [Genome setup — runs once, skipped if files already exist]
    download_genome   Download GRCh38 FASTA from Ensembl
    download_gtf      Download GRCh38 GTF from Ensembl
    index_genome_fa   samtools faidx
    bowtie1_index     Build Bowtie1 index
    ↓
01. FastQC (pre-trim)   QC on raw reads
02. Trim Galore         TruSeq SR adapter trimming (--small_rna)
03. FastQC (post-trim)  QC on trimmed reads
04. Bowtie1             Alignment: -v2 --norc -k10 --best --strata -m100
05. Biotype split       miRNA > tRNA > piRNA > snoRNA > snRNA > rRNA > lncRNA > other
06. SNP filter          3-layer: cell-line + heterozygosity [+ dbSNP]
07. TAPS calling        Pileup → C→T counting → binomial test → BH FDR
    ↓
    [Reports — generated automatically after each stage]
    report_qc           Read length distribution, mapping rates
    report_biotype      Biotype composition
    report_modification Modification rates, top sites, condition comparison,
                        waterfall, trinucleotide context
    report_seqlogos     Sequence logos (±5 and ±10 nt, per biotype)
    multiqc             Aggregated QC HTML report
    ↓
    [Benchmarking — opt-in: snakemake ... all_benchmark]
08. Benchmark*      rastair · asTair · Bismark
09. Compare*        Concordance · Pearson/Spearman correlation
    report_benchmark  Concordance heatmap, correlation, site overlap

* requires all_benchmark target

Genome Setup

Tools: wget, samtools faidx, bowtie-build

If the genome FASTA or GTF do not exist at the paths in config.yaml, the pipeline downloads them from Ensembl automatically. The Ensembl release is set by ensembl_release in config (default: 112). The unmasked dna.toplevel FASTA is always used — the repeat-masked version soft-masks cytosines, which would suppress genuine C→T TAPS modification signal.

If your genome files already exist on disk, all download and indexing steps are skipped. The Bowtie1 index is built once from the FASTA and reused for all samples.

Existing index: Point genome_fa, gtf, and bowtie1_index in config.yaml to your existing files. Snakemake will skip download automatically.

Step 1 — Quality Control

Tool: FastQC, MultiQC

FastQC runs on raw FASTQs before trimming and on trimmed FASTQs after. MultiQC aggregates all results — FastQC, Trim Galore reports, and Bowtie1 alignment logs — into a single interactive HTML report at report/multiqc_report.html. For small RNA libraries, expect a dominant length peak at 18–22 nt (miRNA) and a second peak at 26–32 nt (tRNA fragments and piRNAs). All reads will contain adapter sequence since small RNA inserts are shorter than the read length.

Step 2 — Adapter Trimming

Tool: Trim Galore (Cutadapt)

Trim Galore removes the TruSeq small RNA 3′ adapter (TGGAATTCTCGGGTGCCAAGG) and quality-trims the 3′ end. Reads shorter than 18 nt or longer than 50 nt are discarded. The --small_rna flag sets sensible defaults for this library type.

Step 3 — TAPS-aware Alignment

Tool: Bowtie 1.3.1, SAMtools

Alignment is to the whole GRCh38 genome rather than a transcriptome — tRNA families (~600 gene copies), piRNA clusters, and snoRNA loci need genomic coordinates for correct downstream annotation. Four parameters are set specifically for TAPS data:

ParameterPurpose
-v 2Up to 2 total mismatches — tolerates C→T modifications without penalising them
--norcForward strand only — prevents false G→A calls on reverse strand
-k 10 --best --strataUp to 10 alignments from the best stratum; writes XA:i:N tag for fractional weighting
--samRequired for XA tag output

Step 4 — Biotype Annotation

Tool: Custom Python script (pysam), Ensembl GRCh38 GTF

Reads are assigned to the highest-priority overlapping biotype using the Ensembl GTF, and separate sorted BAMs are written per biotype per sample:

miRNA > tRNA > piRNA > snoRNA > snRNA > rRNA > lncRNA > other

A biotype composition summary (05.biotype_bams/*_biotype_summary.txt) is written alongside each BAM for downstream QC figures.

Note: TAPS enriches for rRNA relative to untreated libraries due to the pyridine borane step. Biotype proportions will differ between conditions — this is expected and not a sign of a failed experiment.

Step 5 — SNP Filtering

Tool: bcftools, custom Python

A C/T heterozygous SNP produces exactly the same signal as a TAPS m5C site. Filtering occurs in three layers before any C→T counts are made:

LayerMethodFlag
1dbSNP common C→T/G→A variants (AF ≥ 1%) — optional, set dbsnp_vcf in configSNP_KNOWN
2Cell-line-specific C→T variants called from merged no-treat BAMsSNP_SAMPLE
3No-treat C→T rate ≥ 40% (heterozygosity proxy)SNP_HET

Only positions with snp_flag == PASS proceed to statistical testing. The snp_flag column is retained in every output TSV for full traceability.

Step 6 — TAPS Methylation Calling

Tool: Custom Python caller (pysam, multiprocessing)

For each cytosine position with sufficient coverage:

mod_rate = T_count / (C_count + T_count)
FeatureImplementation
Strand-specific logicForward strand: modified C reads as T. Reverse strand: modified C reads as A.
CIGAR-aware parsingget_aligned_pairs() handles indels and soft-clipping without positional offsets
Multi-mapper weightingReads with XA:i:N contribute 1/N per locus — prevents inflation at tRNA gene copies and rRNA repeats
Base quality filterBases with Phred Q < 20 are excluded
Parallel processingJobs distributed across chromosomes
Statistical testingBinomial test against background rate, BH FDR correction

Output columns: chrom, start, end, context, mod_count, unmod_count, coverage, mod_rate, pvalue, padj, snp_flag

Step 7 — Background Correction

Tool: Custom Python

δ = treat_mod_rate − pb_Ctrl_mod_rate

Subtracting pb_Ctrl removes both the global pyridine borane background (~8–9%) and sequence-context-specific biases. Sites observed in only one replicate are discarded — stochastic noise is not reproducible across replicates.

Step 8 — Genomic Annotation

Tool: bedtools intersect, Ensembl GRCh38 GTF

Called sites are intersected with the Ensembl annotation to assign gene name, biotype, and feature. miRNA gene names follow miRBase nomenclature. Where a site overlaps multiple features, the most specific annotation is kept using the priority hierarchy from Step 4.

Step 9 — Report Generation

Tools: R (ggplot2, ggseqlogo, BSgenome.Hsapiens.UCSC.hg38), MultiQC

Report figures are generated automatically as part of the pipeline — no manual steps needed. Each report stage runs immediately after its upstream data are ready, so QC figures appear while alignment is still running for the remaining samples.

RuleScriptFigures
report_qc01_qc.RRead length distributions, Bowtie1 mapping rates
report_biotype02_biotype.RBiotype composition per sample; mean by condition
report_modification03_modification.RModification rate distributions, top sites, condition comparison scatter, waterfall, trinucleotide context
report_seqlogos05_sequence_logos.RSequence logos ±5 and ±10 nt per biotype (requires BSgenome)
report_benchmark04_benchmark.RConcordance heatmap, Pearson correlation, site overlap (all_benchmark only)
multiqcMultiQC 1.21Aggregated FastQC + Trim Galore + Bowtie1 HTML report

All figures are saved in PDF, PNG, and SVG at 300 dpi (Arial 8pt, sRNA-TAPS navy-to-teal palette). The R scripts read condition and cell line metadata directly from samples.tsv and adapt colour palettes, axis labels, and condition comparisons to any experimental design — not just HEK/Caco2.

Custom condition labels: Set SRNATAPS_CONDITION_LABELS="treat=TET+PB,pb_Ctrl=PB only,no-treat=Untreated" before running to override display names in figures.

Benchmarking

The optional benchmark suite compares sRNA-TAPS calls against three established tools: rastair (Bowtie1 BAMs, CpG-only and all-context modes), asTair (biotype-split BAMs, all contexts), and Bismark (bisulfite contrast, chemistry inverted for comparison).

# Run benchmark + core pipeline in one command
snakemake \
    --snakefile /path/to/sRNA-TAPS/workflow/Snakefile \
    --configfile config.yaml \
    --profile    /path/to/sRNA-TAPS/profile \
    all_benchmark

This target includes everything in rule all plus rastair, asTair, Bismark, the comparison step, and the benchmark report figures. The benchmark is intentionally opt-in — Bismark genome preparation and asTair on large biotypes are slow and not needed for routine analysis.

asTair environment: asTair must be installed in a dedicated conda environment (astair_env) due to dependency conflicts with Snakemake. The pipeline calls it via its full binary path — no manual activation needed. See the Tutorial for setup.

See the Benchmarking page for full results and interpretation, including the key finding that rastair produces identical output in CpG-only and all-context modes (confirming it is a CpG-only caller by design).

The Delta (δ) Score

δ = treat_mod_rate − pb_Ctrl_mod_rate

Pyridine borane causes ~8–9% non-specific C→T conversion at unmodified cytosines. Subtracting pb_Ctrl at each position removes this background, leaving only the TET-dependent 5mC/5hmC signal.

δ valueInterpretation
~0.00No methylation above background
0.10–0.20Low-level methylation
0.20–0.40Moderate methylation — confident candidate
0.40–0.70High methylation
> 0.70Near-stoichiometric methylation
Note: δ is not absolute stoichiometry. Use it for site discovery and cross-condition comparisons, not as an absolute measure of methylation fraction.

Filtering happens upstream of the binomial test — SNP-flagged positions are excluded entirely. The snp_flag column in every output TSV records the reason for exclusion. Only PASS sites are tested.

Test Dataset

The repository includes a synthetic FASTQ simulator (tests/simulate_taps_srna.py) that generates small RNA reads with realistic TAPS chemistry. Known m5C positions are seeded into the templates at 40–80% conversion in the treat condition and 2–5% background in no_treat.

The test dataset uses 100 samples to stress-test the full pipeline at scale:

python3 tests/simulate_taps_srna.py \
    --outdir ~/srnataps_test/fastq \
    --samples 100 \
    --seed 42

This writes 100 .fq.gz files and a samples.tsv. Reads are 18–50 nt with the TruSeq small RNA adapter. Biotypes: miRNA 40%, tRNA 25%, rRNA 25%, snoRNA 10%, drawn from real hg38 loci.

# Run the test pipeline
cd ~/srnataps_test
ln -sf fastq/samples.tsv samples.tsv

snakemake \
    --snakefile /path/to/sRNA-TAPS/workflow/Snakefile \
    --configfile config.yaml \
    --profile    /path/to/sRNA-TAPS/profile
