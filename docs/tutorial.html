Walkthrough

Tutorial

A complete walkthrough using the 100-sample synthetic test dataset included with sRNA-TAPS. By the end you will have run the full Snakemake pipeline from raw FASTQs to called m5C sites, automatic R report figures, and, optionally, a three-tool benchmarking comparison.

Generate reproducible inputs

Create the synthetic FASTQs, sample sheet, and project structure with the expected chemistry already seeded in.

Launch and monitor the workflow

Move from dry-run validation to full execution, then track progress through SLURM, logs, and output checkpoints.

Verify expected signal

Check seeded miRNA sites, review report figures, and extend the run into optional benchmarking when needed.

Before you start: You need a SLURM cluster with the RNA_taps conda environment active, R packages installed (Rscript install_R_packages.R), and the sRNA-TAPS repository cloned. No pre-built genome index is required — the pipeline downloads the configured species reference automatically.
1

Generate the test FASTQs

The simulator generates 100 samples with accurate TAPS chemistry — seeded m5C positions at 40–80% conversion in treat and 2–5% background in no-treat. Reads are 18–50 nt with the TruSeq small RNA adapter, drawn from real hg38 loci (miRNA 40%, tRNA 25%, rRNA 25%, snoRNA 10%).

python3 tests/simulate_taps_srna.py \
    --outdir ~/srnataps_test/fastq \
    --samples 100 \
    --seed 42

You should see 100 .fq.gz files and a samples.tsv written to the output directory.

Tip: --seed 42 makes the simulation fully reproducible. You will get identical results every time.
2

Set up the project directory

sRNA-TAPS needs two files in the project root: a config.yaml and a samples.tsv. The simulator writes samples.tsv automatically — link it to the project root and create the config.

cd ~/srnataps_test

# Link the generated sample sheet
ln -sf fastq/samples.tsv samples.tsv

Create config.yaml:

project:
  name: srnataps_test
  outdir: /home/user/srnataps_test

reference:
  species:         human
  ensembl_release: 112
  genome_fa:       ""
  gtf:             ""
  bowtie1_index:   ""
  dbsnp_vcf: ""

output:
  fastqc:    02.fastqc
  trimmed:   03.trimGalore
  genome:    04a.genome
  aligned:   04b.aligned
  biotypes:  05.biotype_bams
  snp:       06.snp_resources
  calls:     07.taps_calls
  benchmark: 08.benchmark
  compare:   09.compare
  logs:      logs

benchmark:
  tools:
    astair:
      min_depth: 5
      min_mapq:  20
      min_baseq: 20
      context:   all
Reference setup: This test dataset is human, so the tutorial keeps species: human. Leave genome_fa, gtf, and bowtie1_index empty to let the pipeline download the matching Ensembl reference and build the Bowtie1 index automatically before any samples are processed.
3

Dry-run to verify the DAG

Always run a dry-run first. Snakemake prints the complete list of jobs without executing anything — a clean dry-run means your config, sample sheet, and file paths are all correct.

cd ~/srnataps_test

snakemake \
    --snakefile /path/to/sRNA-TAPS/workflow/Snakefile \
    --configfile config.yaml \
    --profile    /path/to/sRNA-TAPS/profile \
    --dry-run 2>&1 | tail -20

You should see a job count table listing download_genome_fa, download_gtf, bowtie1_index, fastqc, trim_galore, bowtie1_align, biotype_split, taps_call, the four report rules, and multiqc. No errors means you are ready to launch.

4

Run the pipeline

Launch the pipeline via the tmux launcher (recommended for SLURM clusters) or directly in the background. The coordinator process runs on the login node and submits all jobs to SLURM automatically.

Option A — tmux launcher (recommended)

cd ~/srnataps_test
bash /path/to/sRNA-TAPS/sRNA_taps.bash

This opens a named tmux session. Detach at any time with Ctrl+B then D — the run continues in the background. Reattach later with tmux attach -t sRNA_TAPS.

Option B — direct background launch

cd ~/srnataps_test
snakemake \
    --snakefile /path/to/sRNA-TAPS/workflow/Snakefile \
    --configfile config.yaml \
    --profile    /path/to/sRNA-TAPS/profile &
disown %1
disown: Running disown %1 immediately after launch detaches the Snakemake coordinator from your terminal so it survives if you close the session.
5

Monitor progress

Check on the run at any time from a new terminal without interrupting it.

SLURM queue

# Number of active jobs (subtract 1 for the header line)
squeue -u $USER | wc -l

Snakemake log

tail -f ~/srnataps_test/.snakemake/log/$(ls -t ~/srnataps_test/.snakemake/log/ | head -1)

Press Ctrl+C to stop watching the log — this does not affect the running pipeline.

Output checkpoints

# Alignment done?
ls ~/srnataps_test/04b.aligned/*.sorted.bam | wc -l   # expect 100

# TAPS calls done?
ls ~/srnataps_test/07.taps_calls/miRNA/ | wc -l       # expect 100 TSVs

# Report figures generated?
ls ~/srnataps_test/report/figures/*.pdf | wc -l
Reference download first: The first stage downloads and indexes the configured reference genome (~3 GB for human GRCh38). This takes 20–30 minutes and runs once. All subsequent runs in any project on the same cluster reuse the same genome files if you point genome_fa and bowtie1_index to the same paths.

The full 100-sample test run takes approximately 2–4 hours on a standard SLURM cluster, depending on node availability. Report figures for QC and biotype composition appear while later stages are still running.

6

Verify the TAPS signal

Once TAPS calling is complete, check the two seeded m5C positions — chr17:59841313 in hsa-miR-21-5p and chrX:66018955 in hsa-miR-223-3p:

# treat sample — expect mod_rate ~0.90
grep "^17.*59841313" \
    ~/srnataps_test/07.taps_calls/miRNA/treat_HEK_R1_miRNA_taps.tsv

# no-treat sample — expect mod_rate ~0.02
grep "^17.*59841313" \
    ~/srnataps_test/07.taps_calls/miRNA/no-treat_Ctrl_HEK_R1_miRNA_taps.tsv

Expected results:

PositionGeneConditionmod_rate
chr17:59841313hsa-miR-21-5pTET+PB~90%
chr17:59841313hsa-miR-21-5pUntreated~2%
chrX:66018955hsa-miR-223-3pTET+PB~72%
chrX:66018955hsa-miR-223-3pUntreated~0.2%
The >40-fold enrichment between treat and no-treat confirms the pipeline is working correctly end-to-end.

8

Run benchmarking (optional)

The benchmark suite compares sRNA-TAPS calls against rastair (CpG-only and all-context), asTair (all contexts per biotype), and Bismark (bisulfite contrast). It is opt-in because Bismark genome preparation and asTair on large biotypes are slow and not needed for routine analysis.

Set up the asTair environment

asTair has dependency conflicts with Snakemake and must be installed in a separate conda environment. This is a one-time setup:

# Create a dedicated Python 3.9 environment
mamba create -n astair_env -y python=3.9

# Install asTair and setuptools into it
/path/to/conda/envs/astair_env/bin/pip install astair setuptools

# Verify
/path/to/conda/envs/astair_env/bin/astair --help | head -2

Then update benchmark.smk to point to your astair binary path (only needed once):

# In workflow/rules/benchmark.smk, the astair_call rule shell uses:
/path/to/conda/envs/astair_env/bin/astair call \
Note: The pipeline calls asTair via its full binary path — no conda activation is needed at runtime. The path is set once in benchmark.smk.

Launch the benchmark

cd ~/srnataps_test

# Dry-run first
snakemake \
    --snakefile /path/to/sRNA-TAPS/workflow/Snakefile \
    --configfile config.yaml \
    --profile    /path/to/sRNA-TAPS/profile \
    --dry-run \
    all_benchmark 2>&1 | tail -15

# Launch
snakemake \
    --snakefile /path/to/sRNA-TAPS/workflow/Snakefile \
    --configfile config.yaml \
    --profile    /path/to/sRNA-TAPS/profile \
    all_benchmark &
disown %1

The benchmark adds rastair (36 jobs), asTair (144 jobs across 8 biotypes × 18 samples), and Bismark (18 align + 1 index). Monitor with squeue -u $USER. When complete, the comparison outputs and benchmark figures will be at:

~/srnataps_test/09.compare/concordance_summary.tsv
~/srnataps_test/09.compare/correlation_summary.tsv
~/srnataps_test/report/figures/04a_concordance_heatmap.pdf
~/srnataps_test/report/figures/04b_correlation.pdf
~/srnataps_test/report/figures/04c_site_overlap.pdf
Key finding: rastair produces identical outputs in CpG-only and all-context modes — confirming it is a CpG-only caller by design, regardless of flags. This is why a dedicated small RNA pipeline is needed for non-CpG m5C detection. See the Benchmarking page for full results.
